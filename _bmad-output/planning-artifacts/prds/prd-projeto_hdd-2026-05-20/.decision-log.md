### 2026-05-20 — D-035 · Stack v2 Bun-first adoptado
Em `bmad-create-architecture` Step 03, após [A] Bun validation research + [P] 3 perspectivas (DX/Maintenance/Tooling Fit), operador aceitou stack v2 Bun-first.

**Mudanças vs stack v1 (Node):**
- Runtime: Node.js 22 LTS → **Bun 1.3+**
- State store: better-sqlite3 → **bun:sqlite** built-in
- ORM: raw SQL → **Drizzle ORM** (driver bun:sqlite)
- Sandbox: dockerode → **`Bun.spawn('docker', [...])`**
- HTTP client: undici → **fetch nativo**
- Testing: Vitest → **`bun test`**
- Dev runner: tsx → **`bun --hot`**
- Build: tsc → **`bun build`**
- Lint: Biome → **Biome + typescript-eslint** (4 regras async-safety only)
- systemd: 2 units → **1 unit via `litestream run -- bun ...`**
- Secrets: implícito → **envalid/Zod no boot**
- Watchdog: Restart=on-failure → **WatchdogSec=1800 + sd_notify**
- Updates: manual → **Renovate patch-automerge** + ADRs

**Net deps:** -6 runtime (better-sqlite3, tsx, tsc, Vitest, undici, dockerode), +3 (drizzle-orm, envalid, typescript-eslint).

**Tilt-factors:**
1. Anthropic adquiriu Bun em 2026 — infra oficial Claude Code; HDD usa Anthropic SDK → alinhamento estratégico
2. C1 convergência forte: `dockerode → Bun.spawn` resolve principal preocupação Maintenance
3. Plan B Node documentado em 4-6h se Bun falhar (5 ficheiros)

**Riscos aceites:**
- 24/7 daemon stability menos provada (V8 13y vs Bun 2-3y pós-1.0) — mitigação: AO-20 + WatchdogSec + sd_notify
- bun.lockb binário em git review — mitigação: `bun pm whoami` na CI
- Native compat ~34% — improvável bater nas deps escolhidas; trigger para Plan B se aparecer

**Decisão guarda:** memória `project-hdd-stack-v2-bun.md` para futuras sessões.
